# Chrome DevTools Screenshot Smoke Test

Create a static HTML page and use the Chrome DevTools MCP to open it in the browser, take screenshots, and submit them as visual proof.

## Prerequisites

- TaskIt backend running at `http://localhost:8000`
- MCP config generated in the working directory (`odin mcp-setup`)
- TaskIt UI open in browser at `http://localhost:5173`
- `media/` directory writable by the backend process

## Tasks

### Task 1: Create and screenshot a color grid page

Create a file `smoke_output/color_grid.html` — a single static HTML page with inline CSS showing:
- A centered heading: "Chrome DevTools Screenshot Test"
- A 3×3 grid of colored cards, each displaying a color name and its hex code
- Clean sans-serif font, rounded corners, subtle box shadows
- A footer with the text "Generated by Odin — Screenshot Smoke Test"

Then use Chrome DevTools MCP to verify and capture it:
1. Open the file in the browser using `navigate_page` with a `file://` URL
2. Take a text snapshot using `take_snapshot` to confirm the heading, all 9 cards, and footer are present in the page structure
3. Take a full-page screenshot using `take_screenshot` and save it to `smoke_output/color_grid.png`

Submit proof with `screenshot_paths` pointing to the saved screenshot.

**Expected behavior:**
1. HTML file is created with valid markup and inline styles
2. `navigate_page` opens the local file URL in the browser
3. `take_snapshot` returns an a11y tree containing the heading text, 9 color labels, and footer
4. `take_screenshot` produces a PNG file at `smoke_output/color_grid.png`
5. Proof comment includes the screenshot inline in the TaskIt UI
